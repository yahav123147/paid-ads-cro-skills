# עריכת וידאו בעברית (חוצה-פלטפורמה): חיתוך, כתוביות קריוקי, וזום עוקב

צינור מוכח להפוך סרטונים ארוכים לקליפים אנכיים מוכנים לרשת. **כל פרמטר וכל גוצ'ה כאן נולדו מעבודה אמיתית שעבדה.** אל תמציא צינור חדש ואל תתחיל להתקין כלים אחרים, הכל כאן.

**רץ על Windows, macOS ו-Linux.** ההבדל מגרסת המק: התמלול הוא `faster-whisper` (במקום `mlx-whisper` של אפל), ה-setup בפייתון (במקום bash), ו-ffmpeg מגיע מובנה דרך `imageio-ffmpeg`. כל שאר הצינור זהה.

**דיסציפלינת עבודה (חשוב):** קודם מייצרים **פיילוט אחד** ומראים למשתמש, מקבלים אישור סגנון, ואז מייצרים את השאר. המשתמש עובר על הכל ומדווח תיקוני טקסט/חיתוך/צנזור, ומחילים אותם דרך קובצי תיקון ומרנדרים מחדש רק את המושפעים.

## 0. הקמה (פעם אחת)

```bash
python scripts/setup.py     # מתקין faster-whisper + opencv + pillow + imageio-ffmpeg (pip), מאמת מודלים
```
- **Windows:** אם `python` לא מזוהה, נסה `py scripts\setup.py`. צריך Python 3.9+ מ-python.org (סמן "Add Python to PATH" בהתקנה).
- **מק/לינוקס:** אם `python` מצביע ל-2.x, השתמש ב-`python3`.

ה-setup מדפיס `FFMPEG=...` ו-`PY=...`. **אין חובה להגדיר `FFMPEG`** — הסקריפטים מאתרים את ffmpeg המובנה לבד. המודלים (`MobileNetSSD`, `yunet`, `bd.rnnn`) והפונט (`Arial Bold`) **מצורפים בסקיל** תחת `models/` ו-`fonts/`, אז אין הורדות שנחנקות באמצע.

**התמלול (`faster-whisper`)** מוריד בהרצה הראשונה מודל מ-HuggingFace (`large-v3`, ~1.5GB) ומאחסן אותו במטמון. אם הרשת איטית, אפשר לבחור מודל קטן יותר עם משתנה סביבה: `WHISPER_MODEL=medium` (או `small`) לפני ההרצה. `int8` — רץ על כל מעבד, ואם יש GPU של NVIDIA הוא ינוצל אוטומטית.

**איך מריצים סקריפט:** בכל המסמך `python scripts/NAME.py ...` (בווינדוס גם `py scripts\NAME.py ...`). קווים נטויים `/` עובדים גם בווינדוס.

**גוצ'ה לסרטוני אייפון שכבר אנכיים:** `IMG_*.MOV` מאוחסן שוכב עם דגל סיבוב. `-vf` מסובב אוטומטית; `filter_complex` **לא**. לסרטון כזה: לצרוב כתוביות עם `-vf "scale=1080:1920,subtitles=..."`. לניקוי רעש בו-זמנית — `python scripts/denoise.py SRC OUT a.ass` מטפל בזה נכון (מריץ ffmpeg מתיקיית ה-ASS ומעביר את הפונט/מודל בשמות יחסיים, כדי שאות-הכונן `C:` לא תשבור את הפילטר בווינדוס).

## פייפליין A: סרטון מדבר ארוך ⟵ קליפים אנכיים עם זום עוקב

זה הצינור המרכזי (הרצאות, ראיונות, סרטוני דיבור שבהם המצולם **זז**).

### שלב 1 - תמלול וזיהוי רגעים
```bash
python scripts/transcribe.py SRC.mp4 seg.json segments   # מדפיס [mm:ss] טקסט
```
לקרוא את התמלול, לבחור **קטעים חזקים של 40-90 שניות**. אם בסרטון יש 2 קטעים טובים, לפצל לשניים. הוק טוב + עניין + מסר שלם.

### שלב 2 - מעקב פנים על המקור (פעם אחת לכל סרטון מקור)
```bash
python scripts/track.py SRC.mp4 track.json box.jpg 410    # box.jpg = תצוגה מקדימה של תיבת הזום
```
`track.py` מזהה את **הפנים** של המצולם (`YuNet`) ומחזיר טרנספר מיקום + `face_y`. זיהוי טיפוסי 99%+.

### שלב 3 - לכל קליפ: תמלול מילים, כתוביות, וזום-חיתוך
```bash
# א. תמלול הקטע ברמת מילה (חותכים את האודיו של הקטע קודם עם ffmpeg -ss START -t DUR)
python scripts/transcribe.py seg_clip.wav w.json words
# ב. כתוביות קריוקי גדולות + כחול + צנזור
python scripts/captions_hz.py w.json a.ass 1560 140 corr.json  <DUR-0.5>
# ג. זום עוקב + צריבת כתוביות + אודיו (Y0 = face_y - 265)
python scripts/crop_render.py SRC.mp4 START DUR 410 <Y0> a.ass track.json OUT.mp4
```

### פרמטרים מוכחים (פייפליין A)
- **פלט:** אנכי `1080x1920`.
- **זום `CW=410`** (מ-`720` רוחב מקור = פי ~1.8). המסגור עוקב אופקית אחרי הפנים.
- **אנכי `Y0 = face_y - 265`** = מסגור ראש-עד-ירך (לא כל הגוף הקטן, לא חותך ראש).
- **כתוביות `CY=1560`** (שליש תחתון, מתחת לפנים), **`FS=140`** (ענק), הדגשת מילה **כחול** (ברירת מחדל `&HFF901E&`; לצהוב `HIGHLIGHT=&H00E5FF&`, לוורוד `HIGHLIGHT=&HAA37FF&`).
- **הסגנון המאושר (ברירת מחדל):** `MAXWORDS=2 SPACEMULT=0.15` — מקסימום 2 מילים בשורה שמתחלפות מהר, היילייט כחול (ברירת המחדל של הסקריפט), רווח הדוק. `SPACEMULT=0.15` נראה כמו רווח בודד בפועל כי PIL מודד מילים רחב יותר ממה ש-libass מצייר (רווח פנטום); לא להעלות בלי לבדוק פריים. אחרי יצירת ה-ASS להריץ תמיד `python scripts/fix_ltr_lines.py OUT.ass FS SPACEMULT` (מתקן שורות שכולן אנגלית לסדר שמאל⟵ימין).
- מעקב **חלק ויציב** ("פיקס"): `crop_render` מפעיל סינון-חציון + החלקה חזקה על הטרנספר, אז הזום לא קופצני.

**איך מגדירים משתני סביבה לפני פקודה:**
- **מק/לינוקס:** `HIGHLIGHT="&H00E5FF&" MAXWORDS=2 SPACEMULT=0.15 python scripts/captions_hz.py ...`
- **Windows (cmd):** `set "MAXWORDS=2"` ואז `set "SPACEMULT=0.15"` בשורות נפרדות, ואז הפקודה. **Windows (PowerShell):** `$env:MAXWORDS=2; $env:SPACEMULT=0.15; python scripts\captions_hz.py ...`

## פייפליין B: טיזר פודקאסט (מקור אופקי + מוזיקה + כרטיס מותג)

לסרטוני דיבור **סטטיים** (מצלמה קבועה, לא זזים) שהופכים לטיזר אנכי עם מוזיקה וכרטיס סוף מותג.
```bash
python scripts/captions_hz.py w.json a.ass 300 116 corr.json <DUR-0.5>   # CY נמוך יותר, פונט קטן יותר
python scripts/render_teaser.py SRC.mp4 START DUR <CROP_X> a.ass ENDCARD.jpg MUSIC.mp3 <SND_LEN> OUT.mp4
```
- מסגור אנכי **קבוע** `crop=608:1080:CROP_X:0` ממקור `1920x1080` (לכוונן `CROP_X` שהפנים ממורכזות; מצלמה קבועה אז חיתוך אחד).
- **מוזיקה:** משתמשים בכל אורך הסאונד עד לפני השקט (`SND_LEN`, למשל `13.69`), כך שהאינטרו השקט בונה מתחת לסוף הדיבור והביט **נגמר בול** בסוף הסרטון עם כרטיס המותג.
- כרטיס סוף = תמונת מותג ריבועית ממורכזת על שחור, 5 שניות.

## פייפליין C: רב-מצלמתי עם קאטים (2+ דוברים, מצלמה קופצת ביניהם)

לפודקאסט/ראיון שבו **כמה דוברים והעריכה קופצת בין מצלמות סטטיות** (כל מצלמה ממסגרת דובר אחר). כאן פאן חלק **הורס** את החיתוך, הוא גולש לאט על פני קאט חד. הפתרון: חיתוך **מודע-קאטים** שקופץ מיד לדובר החדש בכל קאט.

```bash
# לכל קליפ: 1) תכנון מודע-קאטים  2) אודיו  3) תמלול מילים  4) כתוביות  5) רנדר
python scripts/track_clip.py SRC.mp4 START DUR plan.json      # מזהה קאטים (הפרש פריימים) + פנים דומיננטית לכל שוט
ffmpeg -ss START -t DUR -i SRC.mp4 -vn -ar 16000 -ac 1 seg.wav
python scripts/transcribe.py seg.wav w.json words
python scripts/captions_hz.py w.json a.ass 1540 140 corr.json <DUR-0.4>
python scripts/render_clip.py SRC.mp4 START DUR plan.json a.ass FONTSDIR OUT.mp4 [FACE_MULT]
```

**איך זה עובד:** `track_clip.py` מפענח את הקליפ בפריימים מוקטנים (1280x720, 10fps), מזהה קאט לפי קפיצת הפרש-פריימים, ובוחר לכל שוט את הפנים הדומיננטית (הגדולה בקאט, ואז הקרובה למנעול בתוך השוט, כדי לא לקפוץ בין שני דוברים דומים). הפלט: `shots=[{t0,t1,cx,cy}]` בפיקסלים של המקור. `render_clip.py` נותן לכל שוט **חיתוך סטטי נפרד** ממורכז על הדובר שלו ו-ffmpeg משרשר, כך שהקפיצה בקאט **חדה ומיידית** (בדיוק כמו העריכה המקורית), לא גולשת.

**פרמטרים מוכחים (מקור 4K → ריל):** `FACE_MULT=2.1` → רוחב חיתוך ~`970px` = ראש-כתפיים נעים (גבוה יותר = רחוק יותר; מוגבל אוטומטית לגובה מלא). `CY=1540 FS=140`. הרנדר native לגמרי (בלי פענוח 4K בפייתון) ולכן מהיר.

**באטץ':** `python scripts/batch_reels.py CLIPS.json SRC OUTDIR WORKDIR [FACE_MULT] [CY] [FS]` מריץ את כל 8 השלבים לכל הקליפים ברצף (התמלול לא רץ במקביל). `CLIPS.json`: `[{"name","tag","start","dur","corr":"c_1.json"}]`. **גוצ'ה:** נתיב `corr` נפתר יחסית ל-WORKDIR.

**צנזור:** `captions.py` = צנזור כבוי כברירת מחדל. `captions_hz.py` מצנזר אוטומטית (Meta-safe).

## ניקוי רעש רקע (סרטון מוקלט בחוץ / רועש)
```bash
python scripts/denoise.py SRC.mp4 OUT.mp4 [a.ass]
```
הפחתת רעש מבוססת רשת (`arnndn` עם המודל `bd.rnnn`) + חיתוך רעם נמוך + נרמול. מוריד רעש רקע ב-15-18 דציבל בהפסקות ושומר על הקול. שומר על רזולוציית המקור (קובץ קטן). `mix=0.9` = ניקוי חזק-אך-טבעי (לערוך בקובץ ל-`0.8` לעדין יותר).

## גרסת מהירות (1.2x / 1.5x) — האצה עם שמירת גובה קול

לייצר גרסה מהירה של קליפ **מוגמר** (אחרי שהכתוביות כבר צרובות). מריצים על קובץ הפלט, לא על המקור:
```bash
ffmpeg -y -i OUT.mp4 -vf "setpts=PTS/1.5" -af "atempo=1.5" -r 45 \
  -c:v libx264 -preset medium -crf 20 -pix_fmt yuv420p -c:a aac -b:a 160k -movflags +faststart OUT_1.5x.mp4
```
- **`setpts=PTS/N`** מאיץ וידאו, **`atempo=N`** מאיץ אודיו **בלי לשנות גובה צליל** (בלי אפקט צ'יפמאנק). `atempo` תקין בטווח 0.5-2.0; מעל 2 לשרשר (`atempo=2,atempo=1.25`).
- **`-r` = fps מקורי × N** (30fps → 45 ל-1.5x, 36 ל-1.2x) כדי לא לזרוק פריימים — התנועה נשארת חלקה.
- **כתוביות צרובות מסתנכרנות אוטומטית** כי הן חלק מהפריים ומואצות יחד עם הווידאו. אין צורך לגעת ב-ASS.
- **אימות:** לבדוק שהאורך = אורך מקור / N, ולתפוס פריים על נקודת סנכרון ידועה (מילה שנאמרת) לוודא שהכתובית עדיין יושבת עליה.

## כתוביות קריוקי RTL - הגוצ'ות הקריטיות

עברית + libass + הדגשת מילה = שדה מוקשים. הפתרונות ב-`captions_hz.py`, **אסור לסטות מהם:**

1. **תגי צבע `\1c` בתוך שורה הופכים סדר מילים ב-RTL.** הפתרון: לצייר **כל מילה בנפרד** ב-`\pos` (לבנה תמיד + צבועה בזמן שלה, באותו מיקום). רוחב כל מילה נמדד ב-`PIL`.
2. **כל מילה חייבת עטיפת RLI** (`⁧`...`⁩` = `U+2067`/`U+2069`), אחרת פסיקים ונקודות נוחתים בצד ימין במקום שמאל. `RLM` לבד לא מספיק לפסיקים.
3. **תיקוני טקסט:** לפי אינדקס (`corr`) או לפי מילה מדויקת (`repl`, בטוח יותר, לא תת-מחרוזת). קובץ `corr.json`: `{"corr":{"12":"מילה"},"drops":[13],"repl":{"מזעיפות":"מזייפות"}}`. **חובה לשמור את `corr.json` בקידוד UTF-8** (הסקריפטים קוראים אותו כ-UTF-8, קריטי בווינדוס).
   **שליטה בקיבוץ שורות:** הקיבוץ נשבר על פיסוק בסוף טוקן. כדי ששם מלא יישאר בשורה אחת ("עדן ביבס") — להוסיף פסיק ב-corr לטוקן שלפני השם (כדי לסגור את השורה הקודמת) ולטוקן האחרון של השם. עובד מצוין לרשימות שמות.
4. **הזיות whisper:** על קטע קשה/ארוך whisper ממציא מילים באנגלית/שפה זרה. **לתקן ע"י תמלול מחדש של אותו חלון ב-חתיכות של ~10 שניות** (החלוקה שוברת את לולאת ההזיה). תמיד לסרוק את הטוקנים אחרי אותיות לטיניות זרות.
5. **משפט שלם באנגלית (למשל אאוטרו "Welcome to the NEXT LEVEL"):** הסקריפט מסדר את כל השורות ימין⟵שמאל, אז שורה אנגלית יוצאת הפוכה ("the to Welcome"). הפתרון: `python scripts/fix_ltr_lines.py OUT.ass FS [SPACEMULT]` מיד אחרי `captions_hz.py` — מזהה שורות שכולן לטינית ומחשב להן `\pos` מחדש לסדר שמאל⟵ימין, בשתי השכבות (גם היילייטים של מילים שמתחילות באמצע השורה). שורה מעורבת (מילה אנגלית בודדת בתוך עברית) לא נוגעים — ה-RLI מטפל בה.
6. **מקור אמת מהמסך המצולם:** כשמצלמים דמו של מוצר/פלטפורמה ומילה לא ברורה בתמלול (שם כלי, שם מנחה) — לחלץ פריים ברזולוציה גבוהה מהרגע שבו מדברים עליה ולקרוא את הטקסט מהמסך. עדיף על ניחוש מכל מודל תמלול.

## צנזור מילים שמטא לא אוהב

מוגדר ב-`captions_hz.py` (`CENSOR_SUB`/`WHITELIST`). כל מילה כזו: **הקול מושתק ומוחלף בביפ 1000Hz**, והטקסט **מוסך** (`ס**`). הרשימה הלבנה מונעת חיובי-שווא (`מתחת`, `תחתונים`, `מזייף`, `מזון`). להוסיף/להסיר מילים לפי בקשה. הצנזור נכתב לקובץ `a.ass.ce` ו-`crop_render`/`render_teaser` מיישמים אותו על האודיו.

## גוצ'ות מעקב הזום

- **למרכז על הפנים, לא על מרכז הגוף.** שיער ארוך ויד מושטת מטים את מרכז תיבת-הגוף, והמצולם יוצא לא ממורכז. `YuNet` (פנים) פותר. `MobileNetSSD` (גוף) הוא רק גיבוי כשאין פנים.
- **הפחתת-רקע (`MOG2`) רועשת מדי** למרכוז מדויק, לא לחזור אליה.
- **מעקב חלק = "פיקס".** סינון-חציון + החלקה של 4 שניות נותנים פאן יציב שנראה נעול, לא קופצני.
- **קבצי `track.json` ישנים:** אם שינית פורמט מעקב, למחוק את הישנים לפני הרצה מחדש (אחרת `KeyError`).

## גוצ'ות ווינדוס (חשוב)

- **נתיב כתוביות ב-ffmpeg:** פילטר `subtitles=` נשבר על אות-כונן (`C:\...`) כי הנקודתיים מפרשות כמפריד. **כל סקריפטי הרנדר כאן פותרים את זה** — הם מריצים את ffmpeg מתוך תיקיית ה-ASS, מעתיקים את הפונט לשם, ומצביעים בשמות יחסיים (`subtitles=a.ass:fontsdir=.`). אל תבנה בעצמך פקודת ffmpeg עם נתיב מלא ל-ASS בווינדוס.
- **קידוד קבצים:** כל קובצי ה-JSON עם עברית (`w.json`, `corr.json`) חייבים UTF-8. הסקריפטים כותבים וקוראים UTF-8 מפורש. אם עורכים ידנית ב-Notepad, לשמור כ-UTF-8.
- **ffmpeg:** מגיע מובנה עם `imageio-ffmpeg` (הותקן ב-setup). לא צריך להתקין ffmpeg מערכתי. לפין נתיב אחר: `set "FFMPEG=C:\path\to\ffmpeg.exe"`.
- **GPU:** אם יש כרטיס NVIDIA, `faster-whisper` ינצל אותו אוטומטית ויתמלל הרבה יותר מהר.

## ארגון פלט ותהליך

- קליפים לתת-תיקייה לפי מקור/פרק, שמות ממוספרים `1 - נושא.mp4`.
- אורך קליפ 40-90 שניות. פונט גדול. `9:16`.
- **תמיד:** פיילוט ⟵ אישור ⟵ באטץ' ⟵ המשתמש עובר ומדווח תיקונים ⟵ מחילים דרך `corr.json` ומרנדרים מחדש רק את המושפעים.

## צ'קליסט לפני "מוכן"
- [ ] כל קליפ 40-90 שניות, `1080x1920`, מתחיל ונגמר על משפט שלם (לא באמצע מילה).
- [ ] המצולם ממורכז לאורך כל הקליפ (בדקתי פריימים בהתחלה/אמצע/סוף).
- [ ] כתוביות: פונט גדול, מתחת לפנים, הדגשה כחולה, בלי הזיות אנגלית, פיסוק בצד שמאל.
- [ ] מילים רגישות מצונזרות (ביפ + `ס**`), בדקתי FFT שהביפ ב-1000Hz.
- [ ] טיזרים: מוזיקה נגמרת בול עם כרטיס הסוף.
