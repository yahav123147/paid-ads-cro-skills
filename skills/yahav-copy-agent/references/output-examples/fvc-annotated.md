## Product Profile
| Field | Value |
|-------|-------|
| **Product** | FUNNEL VIBE CODING — סדנה |
| **Type** | 3-day live workshop |
| **Price** | ₪1,297 (direct purchase) |
| **Presenter** | Single presenter (Yahav Rubin) |

| **CTA Type** | Direct purchase button |
| **Angle** | Death Announcement (הודעת פטירה) |

## Why This Example Matters
This is a **mid-ticket, direct-purchase** product. The copy must:
- Create urgency around a NEW technology (Vibe Coding)
- Position against an OLD, dying system (Elementor/drag-and-drop)
- Use a unique creative angle (obituary) to stop scroll
- Price anchor against freelancer costs (₪9,000 vs ₪1,297)
- Single authority figure (not team) = personal story matters more

| NLSS (High-Ticket) | FVC (Mid-Ticket) |

| Application funnel | Direct purchase |

| No price shown | Price shown + anchoring |

## Section-by-Section Annotated Output

### 1. HOOK (Pre-Headline) — "Death Announcement" Angle

```
בצער רב ויגון קודר,
אני מודיע על פטירת עיצוב האלמנטור.
סיום עידן ה"גרירת אלמנטים"
חיסול תקופת ה"טמפלטים המוכנים",
והסוף של כתיבת קופי בעשר אצבעות
ווולקאם לעידן חדש...
```

**WHY THIS ANGLE:**

| Decision | Reasoning |
|----------|-----------|
| **Death announcement format** | Nobody expects an obituary on a sales page. Maximum pattern interrupt. |
| **"Elementor" as the deceased** | Elementor = symbol of the old way, like "Kleenex" for tissues. Everyone knows it. |
| **Escalation structure** | Each line "kills" something else: design → drag-and-drop → templates → manual copywriting. Getting bigger. |
| **"ויגון קודר" (deep sorrow)** | Formal Hebrew = sarcastic tone. The reader knows it's not REAL sorrow. The gap between formal language and actual message = humor. |
| **"ווולקאם לעידן חדש"** | After 5 lines of death → one line of birth. Open Loop: WHICH new era? Must read on. |

**WHY "DEATH ANNOUNCEMENT" AND NOT OTHER ANGLES:**
- The product is about REPLACING an old tool with a new method
- "Death" = the strongest metaphor for "this is over"
- The audience uses Elementor every day — seeing it "die" is visceral
- Works because it's TRUE — vibe coding IS killing the old way
- Would NOT work for: a product that's improving something (not replacing it), or a product with no clear "old way" to kill

**ALTERNATIVE ANGLES CONSIDERED (and why not):**
- "מכתב פרידה מהמעצבת שלי" — Good but less dramatic than death
- "חדשות 2026: 73% מהפאנלים נבנים ללא מעצב" — Good but feels like an ad, not a story
- "מה אם הייתי אומר לך שתוך 72 שעות..." — Decent but too common/expected

### 2. HEADLINE (כותרת ראשית)

```
פאנל מכירה שלם — קופי, עיצוב, אוטומציות ועלייה לאוויר — תוך 72 שעות.
בלי צוות. בלי קוד. רק אתה ו-AI.
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **100% benefit** | Lists what they GET | Not "learn vibe coding" but "get a complete funnel". Outcome, not process. |
| **Specific list** | "קופי, עיצוב, אוטומציות ועלייה לאוויר" | Names exactly what's included. Reader mentally checks: do I need these? Yes to all 4. |
| **72 hours** | Specific timeframe, not "quickly" | "3 days" could work, but "72 hours" sounds more precise/tech. |
| **Triple negation** | "בלי צוות. בלי קוד. רק אתה ו-AI." | Each "without" removes a fear: no team needed, no coding needed. Last line = just you + AI. |
| **"רק אתה ו-AI"** | Frames AI as partner | Not "learn to use AI" but "you AND AI". Partnership feeling. |

**ELEMENTS USED:** Solution (funnel), Specific (72 hours), Simple (no code), Innovation (AI), Timeframe (72 hours)
**ELEMENTS MISSING:** Pain, Proof — these come in the bridge.

**CRITICAL LESSON:** For workshops/courses, the headline should describe the OUTCOME (complete funnel), not the PROCESS (learn vibe coding). The reader doesn't care about the method name — they care about what they'll HAVE at the end.

### 3. BRIDGE (גשר)

```
נעשה את זה קצר.

במשך שנים כתבתי בעשר אצבעות משפכי שיווק שלמים
שהכניסו מעל 100,000,000 ש"ח בשוק הישראלי.

ב-2026 — כל מה שהכרתי השתנה.

היום שבו גיליתי את ה-VIBE CODING —
והרמתי פאנל שלם ב-72 שעות
בלי לכתוב מילה אחת, בלי מתכנת ובלי מעצבת.

ועכשיו?
במשך 3 ימים אני חושף כל מה שאני יודע
בסדנת FUNNEL VIBE CODING.
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **"נעשה את זה קצר"** | Promise of brevity | After the dramatic hook, this says: I respect your time. Let's get to it. |
| **₪100,000,000** | Written in digits, not words | Big number = instant authority. Not "millions" but exact ₪100M+. The + implies it's more. |
| **"כתבתי בעשר אצבעות"** | Callback to hook | "Death of typing with 10 fingers" in hook → "I used to type with 10 fingers" in bridge. He's describing HIS old way too. |
| **"ב-2026 — כל מה שהכרתי השתנה"** | Open loop | Dramatic one-liner. What changed? Must keep reading. |
| **Personal discovery story** | "היום שבו גיליתי" | Makes it a PERSONAL discovery, not a sales pitch. He found something and wants to share. |
| **Workshop teaser** | "במשך 3 ימים אני חושף" | Bridge = give the reader a taste BEFORE diving into pain. They now know something good is coming. |

**CRITICAL LESSON FOR SINGLE-PRESENTER PAGES:**
When there's one presenter (not two), the bridge does DOUBLE duty:
1. Establishes authority (₪100M)
2. Creates the personal "eureka moment" story

For two presenters (like NLSS), this happens in the turning point section. For single presenter, it happens in the bridge because the whole page is more personal.

### 4. LEAD — Pain State (מצב מצוי)

```
אבל קודם, בוא נדבר על מה שקורה היום.

הרעיון לפאנל חדש בוער בראש.
אתה יודע בדיוק מה אתה רוצה לבנות.

אבל בשביל להפוך את זה לדף חי?

צריך לכתוב בריף למעצבת.
לחכות שהקופירייטר יסיים את הפרויקט הקודם.
ולהתחנן בפני המתכנת שיחבר את הטופס לסמוב.

ואז לשבת. ולחכות. ולחכות. ולחכות.

שבוע עובר. שבועיים.
והקופי הכי טוב בעולם — יושב בגוגל דוקס.
כי בלי מעצב ומתכנת, הוא לא שווה כלום.
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **"אבל קודם"** | Controlled pacing | Bridge teased the solution. Now says: wait, let's talk about your reality first. Creates anticipation. |
| **Visual scene** | "הרעיון לפאנל חדש בוער בראש" | SHOWS the moment, doesn't tell. Reader sees themselves in this scene. |
| **Dependency chain** | Designer → copywriter → developer | Three people you depend on. Each one is a bottleneck. The reader counts: 3 people for ONE page?! |
| **"להתחנן"** | Strong verb = emotional | Not "to ask" but "to beg". Hits ego. Business owners don't want to BEG anyone. |
| **"סמוב" (Smoove)** | Named platform | Specificity = credibility. Shows the writer KNOWS this world. Generic = "the email system". |
| **"ולחכות. ולחכות. ולחכות."** | Triple repetition | Reader FEELS the wait. Each repetition = another day wasted. |
| **Google Docs closing** | "הקופי הכי טוב... יושב בגוגל דוקס" | THE most relatable image for this audience. Every marketer has copy sitting in Google Docs waiting for a designer. |

**LEAD TYPE USED: "Problem" (בעיה)**
- Chosen because: the audience is aware of the problem (dependency on freelancers)
- NOT "imagine" because: they don't need to imagine — they LIVE this every day
- NOT "story" because: the bridge already told Yahav's personal story

**UNIVERSAL vs SPECIFIC:**
- Notice: no specific time of day ("11pm"), no specific location
- "בוער בראש" = universal feeling
- "גוגל דוקס" = universally specific (every marketer uses it)
- "סמוב" = specific enough for credibility but common enough that most know it

### 5. LEAD — Dream State (מצב רצוי)

```
עכשיו תדמיין משהו אחר:

אתה קם בבוקר עם רעיון.
פותח את המחשב.
אומר ל-AI בדיוק מה אתה רוצה.

ובצהריים?
הפאנל באוויר.
מעוצב. כתוב. מחובר לדיוור.
ומקבל לידים.

בלי מעצבת. בלי מתכנת. בלי קופירייטר.
רק אתה ו-AI.

זה לא עתידנות. זה מה שאני עושה כל יום.

ובסדנת FUNNEL VIBE CODING —
אני מלמד אותך לעשות את זה בעצמך.
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **"תדמיין"** | Imagination trigger | After pain state, the word "imagine" = relief. Reader's brain switches from problem to possibility. |
| **Morning → afternoon timeline** | "בבוקר... ובצהריים" | Specific daily timeline: idea in AM, live funnel by PM. This is the 72-hour promise made REAL and personal. |
| **Short staccato sentences** | "מעוצב. כתוב. מחובר לדיוור. ומקבל לידים." | 4 one-word/short sentences = speed. The funnel is built FAST. The writing mirrors the speed. |
| **Mirror structure** | "בלי מעצבת. בלי מתכנת. בלי קופירייטר." | Exact mirror of pain state's dependency chain. Same 3 roles, now with "without". Contrast is complete. |
| **"זה לא עתידנות"** | Pre-empties "too good to be true" | Reader is thinking "yeah right". This line says: I know what you're thinking. It's real. I do it DAILY. |

**CRITICAL LESSON:** The dream state must MIRROR the pain state. If pain = 3 dependencies, dream = 0 dependencies. If pain = weeks of waiting, dream = same morning. The contrast should be STRUCTURAL, not just emotional.

### 6. OBJECTION BREAKING (ניפוץ התנגדויות)

```
לא מדובר על עוד טמפלט באלמנטור.
לא על עוד קורס "בנה אתר בלי קוד."
ולא על עוד כלי AI שמייצר טקסט שטוח.

מדובר על שיטה חדשה לגמרי
שמרימה פאנל מכירה שלם — מאפס — תוך 72 שעות.
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **Triple "Not this"** | 3 things it ISN'T | Kills the 3 most common associations: (1) templates, (2) no-code courses, (3) AI text generators. |
| **Named competitors** | "אלמנטור", "AI שמייצר טקסט שטוח" | Specificity. Reader recognizes each one and thinks "yeah, I tried that, it sucked." |
| **"שיטה חדשה לגמרי"** | Category creation | Not an improvement on existing tools — a NEW CATEGORY. This is the "new opportunity" framing from Russell Brunson. |
| **"מאפס"** | One word that matters | From zero = no existing assets needed. Reader doesn't need a website, templates, or content to start. |

**CRITICAL LESSON:** Objection breaking works best in "not this, THIS" format. Kill alternatives first (3 lines max), then introduce yours in 1-2 lines. Ratio should be 3:1 (more killing than introducing). The brain needs to CLOSE old doors before opening a new one.

### 7. SOLUTION REVEAL (חשיפת הפתרון)

```
אני הולך להגיד משהו שיזעזע הרבה אנשים בתעשייה:

היום, אני כבר לא "כותב קוד".
אני גם לא "גורר אלמנטים".
ואני גם לא כותב קופי לדפי מכירה.

אני עושה משהו אחר לגמרי.
משהו חדש.
משהו מפחיד ביעילות שלו.

קוראים לזה:
⚡ VIBE CODING FUNNEL
```

Followed by:

```
מה זה לעזאזל?

עד היום, בשביל להרים פאנל היית צריך 3 אנשים:
מעצב שיעצב.
קופירייטר שיכתוב.
מתכנת שיחבר.

Vibe Coding מוחק את השלושה.

במקום לגרור ווידג'טים או לכתוב שורות קוד —
אתה מדבר ל-AI.

כן, מדבר.

אתה אומר לו:
"תבנה לי דף מכירה. רקע שחור מט.
כפתורים שזוהרים בצהוב.
כל ליד שנרשם מקבל תיוג VIP ברב-מסר
ועובר אוטומטית לדף תודה עם סרטון."

והוא?

בונה את הקוד.
מעצב את הדף.
מחבר את האוטומציות.

תוך דקות.
```

Then proof + compatibility:

```
✅ דף מכירה שנראה כאילו סטודיו עיצב אותו
✅ טפסים שמחוברים אוטומטית לדיוור
✅ דף תודה עם אפסיל
✅ אוטומציות שרצות ברקע

והקטע הגדול?
אפשר לחבר את הפאנל לכל פלטפורמה:
רב מסר, סמוב, שלח מסר, וורדפרס — ועוד.
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **"שיזעזע אנשים בתעשייה"** | Confession/revelation angle | Positions this as INSIDER KNOWLEDGE being shared. Creates tension: what could shock the industry? |
| **Triple negation (again)** | "לא כותב קוד, לא גורר, לא כותב קופי" | Same 3 things from the hook — design, drag-and-drop, copywriting. Full circle. |
| **"מה זה לעזאזל?"** | Street-smart question | Breaks formality. Reader is thinking this exact question. Addressing it = rapport. |
| **Dialogue example** | "תבנה לי דף מכירה. רקע שחור מט..." | SHOWS, doesn't tell. The reader can SEE themselves talking to AI. Specific visual details (black matte, yellow glow) make it concrete. |
| **"כן, מדבר."** | Two-word confirmation | Reader doubts: "talking to AI? Really?" This line says: yes, literally. Confidence. |
| **Compatibility list** | "רב מסר, סמוב, שלח מסר, וורדפרס" | NAMED platforms, not "all platforms". Each name = one less fear ("will it work with MY tools?"). |

**CRITICAL LESSON:** The solution reveal has a 6-part structure:
1. Shocking statement (pattern interrupt)
2. Kill old method (what I DON'T do anymore)
3. Name the solution (give it a name)
4. Simple explanation (how it works in 3 sentences)
5. Proof (screenshots, results, checklist)
6. Compatibility (NAMED platforms, not "everything")

Never skip the compatibility section. The reader's next thought after "cool" is "but will it work with MY stack?" Answer it preemptively.

### 8. OFFER (ההצעה)

```
⚡ FUNNEL VIBE CODING

וזאת הפעם הראשונה – והיחידה – שאני הולך ללמד את זה בלייב באופן אישי.

3 ימים. מעשי לחלוטין.
(זה לא קורס מוקלט. זה אני, אתה, והטכנולוגיה הכי חדשה בעולם.)
```

Day breakdown:
```
יום 1 — מהרעיון לקופי
🔥 איך לגרום ל-AI לכתוב קופי מכירה אמיתי
🔥 איך לבנות הצעה שאי אפשר לסרב לה
🔥 איך להגדיר את ה-Vibe המדויק של הפאנל שלך

יום 2 — מהקופי לפאנל חי
🔥 איך להרים דף מכירה שלם ב-Base44
🔥 איך להוסיף טפסים, סליקה ודפי תודה
🔥 איך לחבר את הפאנל לרב-מסר, סמוב, שלח מסר או וורדפרס

יום 3 — אוטומציות ועלייה לאוויר
🔥 איך לבנות אוטומציות שרצות ברקע
🔥 איך לעלות לאוויר ולהתחיל לקבל לידים — באותו יום
🔥 תחרות: מי שבונה את הפאנל הכי טוב — זוכה בפגישת אסטרטגיה אישית עם יהב
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **"הפעם הראשונה — והיחידה"** | Double exclusivity | "First time" = FOMO. "Only time" = scarcity. Both in one sentence. |
| **"(זה לא קורס מוקלט. זה אני, אתה, והטכנולוגיה)"** | Whisper differentiator | Kills the "just another course" objection in parentheses. Intimate format. |
| **Day titles** | "מהרעיון לקופי" / "מהקופי לפאנל חי" | Each day title is a TRANSFORMATION, not a topic. "From X to Y" = progress. |
| **🔥 emoji** | Fire for content items | Different emoji than other sections (💎=bonuses, ✅=proof, 👈=stats). Visual differentiation. |
| **Competition element** | "מי שבונה את הפאנל הכי טוב — זוכה" | Gamification + bonus incentive. Creates engagement beyond passive learning. |
| **Base44 named** | Specific tool named | Not "a platform" — "Base44". Reader can Google it. Specificity = trust. |
| **3 × 3 structure** | 3 days × 3 bullets each | Clean, scannable, not overwhelming. |

**CRITICAL LESSON (Workshop vs Program):**
- Workshop = day-by-day breakdown (Day 1, Day 2, Day 3)
- Program = phase-by-phase breakdown (Months 1-3, Months 4-6)
- Course = module-by-module breakdown (Module 1, Module 2)

NEVER use months for a workshop or days for a 9-month program. Match the granularity to the product length.

### 9. WHO IT'S FOR (למי מתאים)

```
זה לא לכל אחד.

אם אתה מחפש "כפתור קסם" שיעשה כסף בלי מאמץ —
צא מהדף הזה. ברצינות.

אבל אם אתה:
✦ משווק דיגיטלי שרוצה להרים פאנלים בלי לחכות לאף אחד
✦ יזם שמשלם אלפים למעצב ומתכנת על כל פאנל
✦ בעל עסק שרוצה שליטה מלאה על השיווק שלו
✦ קופירייטר שרוצה להוסיף "בנייה + עיצוב" לשירות שלו

ואתה רוצה לקום בבוקר עם רעיון —
ובצהריים הפאנל באוויר ומקבל לידים?

אתה חייב להיות בפנים.
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **"צא מהדף הזה. ברצינות."** | Aggressive disqualification | PUSHES AWAY "lazy" leads. Counter-intuitive but: rejection = desire. "If they're rejecting people, it must be valuable." |
| **4 avatars** | Marketer, entrepreneur, business owner, copywriter | Each ✦ = a specific person. Reader sees themselves in ONE of these. |
| **"קופירייטר שרוצה להוסיף בנייה + עיצוב"** | Niche expansion | Most sales pages forget this avatar. Copywriters who can also BUILD = higher value. Shows Yahav thinks about THEIR business. |
| **Dream callback** | "לקום בבוקר... ובצהריים הפאנל באוויר" | Exact same language as the dream state section. Repetition = reinforcement. |
| **"אתה חייב להיות בפנים"** | Imperative, not invitation | Not "consider joining" but "you MUST be inside." Urgency through command. |

### 10. PRICING + ANCHORING (תמחור)

```
בוא נעשה חשבון:

👈 מעצב פרילנסר: ₪3,500 לדף אחד.
👈 קופירייטר: ₪4,000 לפאנל.
👈 מתכנת לחיבורים: ₪1,500.
👈 סה"כ פאנל אחד: ₪9,000 + שבועיים המתנה.

ואתה עדיין תלוי בהם בפאנל הבא.

כאן אתה מקבל את היכולת לבנות פאנלים בלי הגבלה. לכל החיים.

✅ 3 ימי סדנה מעשית לייב עם יהב רובין
✅ הקלטות מלאות של כל הסדנה
✅ גישה לקהילה + תמיכה שוטפת
✅ תבניות Vibe Coding מוכנות להעתקה

המחיר שלך היום:

₪1,297 בלבד.

(כולל הבונוסים. כולל ההקלטות. כולל אחריות מלאה.)
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **"בוא נעשה חשבון"** | Invites reader to calculate | Logical framing. Not "we're cheap" but "let's do the math together." |
| **Freelancer cost breakdown** | ₪3,500 + ₪4,000 + ₪1,500 = ₪9,000 | REALISTIC anchor. Not inflated. Reader thinks "yeah, that's about right." Believable anchor = effective anchor. |
| **"+ שבועיים המתנה"** | Adds TIME cost | Money + time = double anchor. Even if they have ₪9K, the WAIT is painful. |
| **"ואתה עדיין תלוי בהם"** | Recurring cost reframe | ₪9K per funnel. Not one-time. Every new funnel = another ₪9K. Workshop = unlimited funnels forever. |
| **"בלי הגבלה. לכל החיים."** | Infinity framing | One payment → unlimited output. The math is irresistible: ₪1,297 / infinity funnels. |
| **✅ list** | 4 clear inclusions | Not 17 items. 4 concrete things. Scannable. Each has clear value. |
| **"₪1,297 בלבד"** | Price + "only" | "בלבד" (only) reframes the price as small. After ₪9K anchor, ₪1,297 feels like nothing. |
| **Parenthetical** | "(כולל הבונוסים. כולל ההקלטות. כולל אחריות מלאה.)" | 3 inclusions in whisper format. Each "כולל" = one less worry. |

**CRITICAL LESSON (Pricing for mid-ticket):**
- Anchor = realistic alternative cost (freelancers), NOT inflated "value"
- Show anchor as ITEMIZED breakdown (not just "worth ₪9,000")
- Add TIME cost to money cost
- Emphasize RECURRING vs ONE-TIME nature
- Price must feel like a fraction of the anchor

### 11. CTA + URGENCY (דחיפות)

```
המתחרים שלך עדיין בונים בשיטה הישנה.
עדיין מחכים למעצבת.
עדיין משלמים ₪9,000 על פאנל אחד.

יש לך חלון הזדמנויות צר
ללמוד את השיטה לפני כולם.

[CTA: אני בפנים! שריינו לי מקום בסדנה]
(מספר המקומות מוגבל | גישה מיידית לחומרים | 100% אחריות)
```

**WHY IT WORKS:**

| Element | Decision | Reasoning |
|---------|----------|-----------|
| **Competitor framing** | "המתחרים שלך עדיין..." | Urgency through competitive advantage. Not "buy before price goes up" but "learn before competitors do." |
| **₪9,000 callback** | Repeats anchor one more time | Last reminder of the alternative cost before the button. |
| **"חלון הזדמנויות צר"** | Window metaphor | Not "limited time" (cliché) but "narrow window of opportunity" (specific, visual). |
| **CTA text** | "!אני בפנים! שריינו לי מקום" | First person ("I'm in!") + action ("save my spot"). Exclamation = energy. |
| **3 items under button** | Scarcity + instant access + guarantee | Removes 3 final objections: (1) will spots run out? (2) when do I get access? (3) what if I don't like it? |

**CRITICAL LESSON (CTA for direct purchase vs application):**
- Direct purchase: "אני בפנים! שריינו לי מקום" = action + excitement
- Application: "רוצה לבדוק אם אני מתאים" = curiosity + exclusivity
- NEVER use generic "הזמן עכשיו" or "קנה עכשיו" — always first person + specific action

## Key Patterns to Replicate (Mid-Ticket Workshop)

1. **Creative angle = pattern interrupt** — Death announcement, not generic "workshop with Yahav"
2. **72-hour specificity** — Exact timeframe repeated throughout (hero, bridge, headline, offer)
3. **3 dependencies = 0 dependencies** — Pain has 3 people you need → Solution needs 0
4. **Freelancer cost anchoring** — Realistic, itemized breakdown beats inflated "value"
5. **Day-by-day offer** — Workshop = daily breakdown (not modules or phases)
6. **"Not this, THIS"** — Kill 3 alternatives before introducing yours
7. **Dream state mirrors pain state** — Same structure, opposite outcome
8. **Named tools** — "Base44", "סמוב", "רב מסר" = credibility through specificity
9. **Single presenter = personal story** — Bridge carries more of the authority (no 2nd founder to share it with)
10. **Competition element** — Gamification adds engagement to workshop format

## How FVC Differs from NLSS Pattern

| Pattern | NLSS | FVC |
|---------|------|-----|
| **Hook** | "ביי ביי" farewell list | Death announcement (הספד) |
| **Bridge** | Short authority + open loop | Personal discovery story |
| **Lead** | 2 bad options (broad) | Dependency chain (specific) |
| **Turning point** | Separate section with origin story | Embedded in bridge (single presenter) |
| **Authority** | 2 founder profiles | Single presenter credentials in bridge |
| **Solution** | System with 3 engines + phases | Tool demo with dialogue example |
| **Offer** | Phase-based (months) | Day-based (3 days) |
| **Pricing** | No price shown (application) | Price shown with anchor |
| **CTA** | "בדוק אם אתה מתאים" | "שריינו לי מקום" |
| **Scarcity** | 15 spots, 50% rejected | Limited spots + window of opportunity |
